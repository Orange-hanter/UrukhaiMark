# Runbook оператора

> Ежедневные операции маркировки через UrukhaiMark (после внедрения).

## 1. Ежедневный чеклист

- [ ] Проверить остаток неиспользованных КМ (≥ 7 дней запас)
- [ ] При необходимости — заказать новую партию кодов
- [ ] Проверить работу принтера (тестовая этикетка)
- [ ] Убедиться, что token/API доступен (health green)

## 2. Стандартный процесс: партия на экспорт РФ

1. **Создать партию** — выбрать товар (GTIN), количество, даты производства/годности
2. **Заказать коды** — дождаться статуса «коды получены»
3. **Печать** — запустить печать этикеток, проверить первые 3 шт. сканером/приложением
4. **Маркировка** — нанести этикетки на товар
5. **Отчёты** — отправить «маркировка» → «производство»
6. **Отгрузка** — указать ТТН, контрагента, сертификаты → отправить
7. **Контроль** — убедиться, что контрагент РФ видит поставку

## 3. Проверка DataMatrix

1. Открыть приложение «Электронный знак»
2. Сканировать этикетку
3. Если ошибка — **не** продолжать партию; см. [troubleshooting.md](../troubleshooting.md)

## 4. Что делать при ошибках

| Ситуация | Действие |
|----------|----------|
| Заказ завис | Подождать 30 мин; проверить status в UI; support@datamark.by |
| Печать смазана | Перепечатать с новыми кодами (если не submitted) |
| addMark failed | Не отправлять manufacture; проверить GS |
| ship rejected | Проверить сертификаты и manufacture report |
| API недоступен | Fallback: [ЛК datamark.by](https://datamark.by/) — ручной режим |

## 5. Fallback — ручной режим (ЛК)

Если UrukhaiMark недоступен:

1. Войти в ЛК «Электронный знак»
2. Заказ кодов → скачать txt
3. Печать через «Электронный кабинет» или сторонний генератор GS1 DataMatrix
4. Отчёты и отгрузка — через ЛК

## 7. Запреты

- Не копировать КМ в Excel
- Не использовать QR-генераторы вместо GS1 DataMatrix
- Не отправлять отгрузку без manufacture report
- Не смешивать товарные группы в одной отгрузке
