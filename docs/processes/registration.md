# Регистрация и подключение

Чеклист выполняется **один раз** перед первым заказом кодов.

## 1. GS1 Бел

- Сайт: [gs1by.by](https://gs1by.by/)
- Получить **GCP** (префикс компании) и **GLN**
- Без GCP нельзя регистрировать GTIN

## 2. ePASS — электронные паспорта

- Сайт: [epass.by](https://epass.by/)
- Описать каждый товар (наименование, ТН ВЭД, характеристики)
- Получить **GTIN** (14 цифр; в API часто с ведущим `0`)

## 3. ГИС «Электронный знак»

- [Заявка на подключение](https://datamark.by/)
- Принять публичный договор (`is_rules_agree: true` при первом `/auth`)
- Получить логин/пароль ЛК

## 4. Товарные группы

Отправить форму на **support@datamark.by** (на фирменном бланке):

| Группа | Для чего |
|--------|----------|
| `cosmetics` | Освежители → экспорт РФ |
| `ukz` | Пиво → продажа РБ |

## 5. API для UrukhaiMark

Заявка integrator на support@datamark.by:

- УНП
- GLN
- Скан свидетельства о регистрации

После одобрения:

| Контур | URL |
|--------|-----|
| Sandbox | `https://sandbox-api.datamark.by` |
| Production | `https://api.datamark.by` |

## 6. Контрагент в РФ

- Регистрация в «Честном знаке»
- Готовность принимать отгрузки из datamark.by
- Для пива: возможность заказа кодов CRPT

## 7. Синхронизация каталога

1. GTIN создан в ePASS
2. GTIN добавлен в ЛК «Электронный знак» или через `POST /items`
3. Проверка: заказ тестового КМ в sandbox

## Типичные ошибки регистрации

| Ошибка | Решение |
|--------|---------|
| `Business entity not found: UNP, GLN` | Проверить регистрацию и GLN |
| `GTIN is not found` | Синхронизация ePASS → каталог |
| `GTIN не синхронизирован` | Подождать или обратиться в поддержку |
